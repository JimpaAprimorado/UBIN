Contribuindo para Protocolo Fundamental Essencial

Obrigado por considerar contribuir com este projeto!
Para manter a organização e integridade do código, siga as diretrizes abaixo.

Como contribuir
1. Abra uma Issue

Antes de iniciar qualquer mudança, crie uma issue explicando:

O problema ou melhoria desejada

Qual parte do projeto será afetada

Qual o impacto da mudança

Isso evita trabalho duplicado ou contribuições fora do escopo.

Fluxo de Trabalho (Workflow)

Faça um fork do repositório.

Crie um ramo (branch) descritivo:

git checkout -b feature/nome-da-funcionalidade


Faça commits pequenos, claros e diretos.

Envie um Pull Request (PR) para o branch main.

Padrões de Código

Siga a organização estabelecida no repositório.

Não inclua arquivos temporários, caches ou dependências não essenciais.

Não insira chaves, tokens, ou informações sensíveis.

Comente funções complexas.

Quando possível, utilize mensagens de commit descritivas, por exemplo:

Melhora rotina de parsing binário
Corrige falha no interpretador
Refatora módulo de criptografia

Segurança

Se encontrar uma vulnerabilidade, não abra issue pública.

Siga o procedimento descrito no SECURITY.md.

Testes (se aplicável)

Inclua testes para novas funcionalidades.

Certifique-se de que todos os testes existentes continuam passando.

Não quebre compatibilidade sem justificativa.

Ao enviar um Pull Request

Confirme:

A alteração foi testada

Não há informações sensíveis no commit

O propósito da mudança está explicado

A documentação foi atualizada (se necessário)

Contato

Para questões delicadas, utilize comunicação privada com o mantenedor do repositório.Como contribuir
